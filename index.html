Abstract

This ontology provides a structured semantic framework for selecting and applying knowledge cleaning approaches for cleaning Knowledge Graphs. It defines key concepts, relationships, and dimensions relevant to knowledge cleaning, including:

  • Knowledge Graph Features
  • Knowledge Cleaning Approaches & Techniques
  • Dimensions of Approaches
  • Background Knowledge
  • Application Algorithms

Designed to facilitate the selection of optimal cleaning methods, the ontology answers a central question:

Given a Knowledge Graph, its properties, cleaning sources, and a specific error type, what are the most suitable approaches for detecting and correcting the errors?

Additionally, it supports subordinate questions regarding:

  • Specific error types
  • Approach dimensions
  • Cleaning source compatibility
  • Background knowledge required for each approach

The ontology further defines semantics for applying the approaches through application algorithms and logic.

By providing well-defined semantics, the ontology enables the implementation and application of rules to construct effective knowledge cleaning pipelines, ensuring error detection, correction, and validation in Knowledge Graphs.

Introduction back to ToC

The Holistic Knowledge Cleaning Ontology (HKCO)

The Holistic Knowledge Cleaning Ontology (HKCO) provides the semantics for knowledge cleaning. At its core, the ontology includes two key components: the knowledge graph and the cleaning approaches. Its primary goal is to enable the selection of appropriate approaches for detecting and correcting errors within a knowledge graph.

Error Representation

Errors are specified through two dimensions:

  • Error Source: The specific type of assertion affected by the error.
  • Error Nature: The nature of the error, which can be syntactic or semantic.
These two dimensions combine to define a specific error type. For example, an instance assertion might contain a spelling mistake in the type name.

Knowledge Cleaning Approaches

Knowledge cleaning approaches implement cleaning techniques that define how sources are processed. The ontology distinguishes between two types of sources:

  • Internal Sources: The knowledge graph and its assertions.
  • External Sources: Background knowledge such as the Web, other databases, or structured documents.
The choice of source is a key dimension of an approach or technique, and the ontology provides semantics to represent and relate these dimensions.

Quality Specifications

Sources—whether internal or external—vary in quality, which includes aspects like completeness and correctness. The ontology provides semantics to define quality specifications for sources. These specifications influence the dimensions of an approach and help determine the best-suited cleaning techniques based on the quality features of available sources.

Application of Knowledge Cleaning Approaches

The ontology also represents the application process of selected approaches. This process involves:

  • Defining logic steps, tasks, and conditions.
  • Structuring a cleaning pipeline using application algorithms.
  • Ensuring correctness of predicted results through additional validation and error detection processes.

Central and Subordinate Questions

The ontology is designed to answer one main question:

How to select and apply knowledge cleaning approaches, considering a knowledge graph, user requirements, and available sources?
It also supports subordinate questions, such as:
  • Which approach targets *** error?
  • Which approaches have *** dimension?
  • Which cleaning techniques support *** background knowledge?
  • What type of background knowledge does technique *** use?
  • What are the quality specifications for *** type of background knowledge?

Conclusion

By providing well-defined semantics for errors, approaches, techniques, quality specifications, and application algorithms, the HKCO enables a structured and systematic knowledge cleaning process. It ensures error detection, correction, and validation within knowledge graphs, making it a valuable resource for effective knowledge cleaning pipelines.

HolisticKnowledgeCleaningOntology: Description back to ToC

The holistic knowledge cleaning ontology represents important features of knowledge cleaning. These features include knowledge cleaning approaches, techniques and backgroundknowledge. It aims to provide the context from which knowledge cleaning can be performed. The ontology proviedes means to select approaches for a custom cleaning process, depending on for example, the knowledge graph and available additional cleaning sources. After the selection the approaches need to be applied, this application is represented by the Application algorithm and logic classes.
